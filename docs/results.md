# Results files

Each run of `python scripts/run_training.py` (or the notebook) writes
the artifacts below.

## Per-model figures (`figures/`)

* `fig1_confusion_matrix_<short>.png` — pooled confusion matrix with
  cell counts and percentages.
* `fig2_fold_distribution_<short>.png` — per-fold accuracy / macro-F1 /
  ROC-AUC scatter with mean and ±1 SD band.
* `fig3_roc_curve_<short>.png` — per-fold ROC curves with the mean and
  a ±1 SD band.
* `fig4_learning_curve_<short>.png` — eval loss / accuracy / F1 trace
  across training epochs.

## Cross-model figures (`figures/`)

* `fig_cross_model_comparison.png` — three-metric bar chart with
  paired-*t*-test outputs printed to stdout.
* `fig_roc_overlay.png` — pooled ROC for every backbone, overlaid.
* `fig_paired_per_fold.png` — per-fold accuracy connected across
  models (paired by fold index).
* `fig_learning_curves.png` — eval-loss trajectories overlaid for the
  three backbones.

The notebook produces both PNG (default) and SVG variants. The CLI
script `scripts/generate_figures.py` writes PNGs to match the file
names above.

## Canonical aliases referenced in the report

* `fig1_confusion_matrix.png`, `fig5_roc_curve.png`,
  `fig8_mean_metrics.png` — copies of the most representative
  per-model / cross-model PNGs, cited directly in the final report.
* `fig10_dataset_distribution.png`, `methodology_figure.png` —
  generated by `scripts/generate_figures.py` from the dataset and the
  pipeline diagram in `src/plotting.py`.

## Tables (`tables/`)

* `per_fold_<short>.csv` — per-fold metric table for one backbone.
* `results_<short>.json` — full predictions + summary stats for one
  backbone (heavy file).
* `cross_model_comparison.csv` — side-by-side `mean ± std` comparison
  across backbones.
* `cross_model_results.json` — combined summary (drops the heavy
  per-sample arrays; those live in the per-model JSON files).
* `summary_metrics.csv` — one-row-per-backbone headline numbers.
* `statistical_tests.csv` — paired-*t*-test results across every
  metric × pair (matches Section 11 of the notebook).
* `model_comparison.csv` — pretty-printed comparison table.
* `fold_metrics.csv` — long-format per-fold metrics for every
  backbone × seed × fold.

## Per-run artifacts (`results/`)

* `predictions_<short>.csv` — one row per record with true / predicted
  label and class-probability for one backbone.
* `fold_metrics.csv` — long-format per-fold metrics (mirrors
  `tables/fold_metrics.csv`).
* `final_summary.json` — aggregate metrics + paired-*t*-test array,
  used as the canonical machine-readable summary of the run.

## Generating outputs without training

If you want to populate the folders without paying the GPU cost, run::

    python scripts/generate_sample_outputs.py

This drives the plotting and aggregation code with realistic synthetic
predictions so you can see the intended layout. Real training results
will overwrite the sample files when you run
`python scripts/run_training.py` or execute the notebook end-to-end.
